# knowledgevault 0.6.1 Release Notes

Release date: 2026-02-17
Package: `knowledgevault==0.6.1`
Type: Security patch

## Summary

This release fixes MCP path validation gaps that could allow path traversal outside the configured knowledge base root for specific write/move operations.

## Included Fixes

1. `kvault_write_summary`
- Rejects paths that escape KB root.

2. `kvault_move_entity`
- Validates both `source_path` and `target_path`.
- Rejects any source/target path that escapes KB root.

3. Regression tests
- Added explicit traversal tests for:
  - summary write
  - entity move (source and target)
  - batch summary update

## Impact

- Existing valid workflows are unaffected.
- Invalid/traversal-like paths now return structured validation errors instead of writing/moving files.

## Deployment

1. Upgrade package:
```bash
python3.12 -m pip install --upgrade "knowledgevault==0.6.1"
```

2. Restart any service manager or gateway that spawns `kvault-mcp` processes:
```bash
your-service restart
your-service health
```

3. Quick verification:
```bash
python3.12 - <<'PY'
from kvault import __version__
print(__version__)
PY
```
Expected output:
`0.6.1`

## Rollback

If needed:
```bash
python3.12 -m pip install "knowledgevault==0.6.0"
your-service restart
```
